## Recommendations — status update (2026-07, second pass)

- ✅ **Rate-limit `/api/assessment/submit` + `/questions`**: done —
  `src/lib/rateLimit.ts` (in-memory sliding window, per IP; submit 5/hr,
  questions 30/hr). Known limitation: per-serverless-instance state; upgrade
  to Vercel WAF / Upstash if real abuse appears.
- ✅ **Test suite**: done — vitest with 15 tests over
  `answerUtils` (fraction/又 normalization, grading) and
  `mockExamSelection` (section counts, group atomicity ×100 runs, no dupes,
  sparse pools). `npm test`, wired into CI before the build step.
- ✅ **Signup email rate limit**: root-caused — Supabase's shared SMTP caps at
  a handful of emails/hour (`over_email_send_rate_limit`), which broke signup
  entirely. Signup now uses `auth.admin.createUser({ email_confirm: true })`
  (no email sent) + direct sign-in. If the academy configures custom SMTP
  later and wants verification emails, switch back to `signUp()` — the
  template in `supabase/email_templates/` is ready. The double-submit race is
  mitigated by the pending-disabled submit button (useFormStatus).
- 📋 **Legacy `questions`/`question_categories` consolidation**: still
  deferred — touched by wrong-question bank + variations flow; a proper
  migration onto `assessment_questions` is its own project (see CLAUDE.md
  「雙表並存」note).
